Darling is a toolkit for Go that aims to provide simple, modular packages that are light abstractions on top of the Go standard libraries. Unlike the rest of Impractical Lab’s projects, Darling is built with the community in mind, and we will do what we can to support use cases outside our own.

A project becomes a Darling project when we feel we’ve solved it in a general enough way that it would be useful to other, not-us people.
